# 星みちTRPG キャラクタービルダー 採用ルール集

この文書は、キャラクタービルダー制作中の会話で決定し、現在のサイトへ実装した内容をまとめたものです。

- 対象：キャラクタービルダーの入力、計算、表示、出力、KPモード
- 位置付け：キャラクタービルダーで現在採用している仕様
- 注意：星みちTRPGのルール正本そのものではありません。正本へ反映する場合は別途確認します。

## 最重要：記号の読み方

### CP・HP・AP・正のBLD

| 記号 | 数値 |
|---|---:|
| `★` | 5 |
| `☆` | 1 |

`★`は5ポイント分、`☆`は端数の1ポイント分です。

```text
★ ＝ 5
★☆ ＝ 6
★★ ＝ 10
★★☆☆ ＝ 12
```

記号が長くなったときは、**記号5個ごとに半角スペース**を入れます。スペース自体に数値はありません。

| 数値 | 表示 | 数え方 |
|---:|---|---|
| 6 | `★☆` | 5＋1 |
| 12 | `★★☆☆` | 5＋5＋1＋1 |
| 25 | `★★★★★` | 5×5 |
| 27 | `★★★★★☆☆` | 5×5＋1＋1 |
| 40 | `★★★★★ ★★★` | 5×8 |
| 67 | `★★★★★ ★★★★★ ★★★☆☆` | 5×13＋1＋1 |

`★★★★★ ★★★`は「星が8ポイント」ではなく、`★`が8個あるため**40ポイント**です。

### SP・MP・FP・割り振り能力

| 記号 | 数値 |
|---|---:|
| `■` | 1 |

```text
■ ＝ 1
■■ ＝ 2
■■■■■ ■■ ＝ 7
```

こちらも、記号5個ごとに半角スペースを入れます。

### BLDの0と負数

正のBLDはHPやAPと同じ星表示です。0以下だけ三角で表します。

| BLD | 表示 |
|---:|---|
| 2 | `☆☆` |
| 1 | `☆` |
| 0 | `△` |
| -1 | `△△` |
| -2 | `△△△` |
| -3 | `△△△△` |

負のBLDは短縮しません。`△`がBLD 0で、そこから1下がるごとに`△`を1個追加します。

```text
△ ＝ BLD 0
△△ ＝ BLD -1
△△△ ＝ BLD -2
```

## キャラクターシート要約

要約は次の順番で自動生成します。

1. 名前
2. 武器
3. 防具
4. スキル
5. CP
6. HP
7. AP
8. BLD
9. SP
10. MP
11. FP
12. 割り振り能力

例：

```text
## キャラクターシート要約

名前：アルタ
武器：鉄の剣【7】　短剣【4】
防具：旅装【2】
スキル：突進【35】

CP：★★★★★ ★★★
HP：★★★★★ ★★★★★ ★★★☆☆
AP：★★★★★☆☆
BLD：☆
SP：■■

割り振り能力
頑健：■
持久：■■■■■
筋力：■■■
技量：■■■■
敏捷：■■
感覚：■
精神：■
料理：■
手当：■
冒険：■
```

要約の表示規則：

- CP・HP・AP・正のBLDは`★＝5`、`☆＝1`
- BLD 0は`△`
- 負のBLDは、0の`△`に負の段階ぶん`△`を追加
- SP・MP・FP・割り振り能力は`■＝1`
- BLDは必ずAPの直下
- BLD以外の0の資源・能力は表示しない
- 名前のない武器・防具・スキルは表示しない
- 武器が2本ある場合は番号を付けず、同じ行へ横並び
- 武器が1本だけなら、その1本だけを表示
- モンスターの要約には実務技能を表示しない

## 略称

| 略称 | 意味 |
|---|---|
| CP | キャラクターポイント／総合ポイント |
| HP | ヒットポイント |
| AP | スタミナ |
| SP | 気力 |
| MP | 魔力 |
| FP | 聖力 |
| BLD | 体格。クトゥルフのビルドと同義 |

BLDは`SIZ`ではありません。このゲームの体格は「大きさ」よりも「ビルド」に相当するため、`BLD`を採用します。一般的な人はBLD 1です。

## 作成対象

入力画面には「作成対象」の切替があります。

### キャラクター

- 実務技能を表示
- 実務技能をポイント計算へ含める

### モンスター

- 実務技能を非表示
- 実務技能をポイント計算から除外
- 切替前に入力した実務技能値は削除せず保持
- キャラクターへ戻すと、保持していた実務技能を再表示

料理、整備、手当、植物、鉱物、解体、冒険はモンスターには通常使用しません。

例外的に薬草調合や鍛冶を行うモンスターは、実務技能ではなく個別のスキルとして登録します。

```text
粗雑な薬草調合
武器修繕
```

特に「冒険」はプレイヤーとチャッピーの共同表現へ作用する技能なので、NPCやモンスターには使用しません。

## キャラクターポイント

- 合計ポイントの初期値：4
- 入力範囲：0～999
- 能力を1上げるごとに1ポイント消費
- 使用ポイントが合計ポイントを超えた場合は警告
- ポイント超過中はプレイヤー用・チャッピー用・KP用Markdownをコピー不可
- 合計ポイントを後から減らしても、割り振り済み能力は自動削除しない

キャラクターの使用ポイント：

```text
基礎ステータス合計
＋ 潜在能力合計
＋ 実務技能合計
```

モンスターの使用ポイント：

```text
基礎ステータス合計
＋ 潜在能力合計
```

合計ポイントの指標：

| CP | 指標 |
|---:|---|
| 0 | ポイント未設定 |
| 1～4 | 駆け出し |
| 5～19 | 成長中 |
| 20～29 | 熟練者 |
| 30～39 | 銀級相当 |
| 40 | 銀級上位 |
| 41以上 | 銀級上位以上 |

シーリスは銀級40ポイントの指標ですが、銀級の中でも上位として扱います。サイト上では固有名を表示せず「銀級上位」と表示します。

## 潜在能力

| 能力 | 関わる内容 | 才能の目安 | 上限 |
|---|---|---|---:|
| 気力 | 肉体強化、武技 | みんな持ってる。正しく使えるのは一握り | 10 |
| 魔力 | 魔術、錬金術 | 10人にひとりの才能 | 10 |
| 聖力 | 治癒、浄化、加護 | 100人にひとりの才能 | 10 |

表示：

- 0：`―`、使用不可
- 1：習得数1・1日1回
- 2：習得数2・1日2回
- 以下、値と同数
- 10：習得数10・1日10回

潜在能力値は、対応スキルの「習得可能数」と「一日の使用可能量」を兼ねます。

- 気力0：一般基準を超えるポイントを割り振っていない状態
- 魔力0・聖力0：人物正本に例外がない限り、使用可能な才能が確認されていない状態
- 習得済みスキル数より潜在能力を低くできない
- 設定済みのスキル消費より潜在能力を低くできない

## 実務技能

| 技能 | 上限 |
|---|---:|
| 料理 | 3 |
| 整備 | 3 |
| 手当 | 3 |
| 植物 | 3 |
| 鉱物 | 3 |
| 解体 | 3 |
| 冒険 | 5 |

通常の実務技能の段階：

| 値 | 表示 |
|---:|---|
| 0 | 未習得 |
| 1 | 心得あり |
| 2 | 得意 |
| 3 | 熟練 |

冒険4・5には固定の段階名を付けません。

KP内部計算では、通常の実務技能補正は次の式です。

```text
実務技能補正 ＝ 技能値×20
```

冒険は共同表現へ作用するため、通常の成功値加算には使用しません。

## HP・AP・BLD

### HP

キャラクター：

```text
最大HP ＝ round（50×（1＋頑健÷3））
```

モンスター：

- 最大HPを個別入力
- 初期値25

### AP

```text
最大AP ＝ round（10×（1＋持久÷3））
```

例：

| 持久 | 最大AP |
|---:|---:|
| 0 | 10 |
| 1 | 13 |
| 5 | 27 |
| 15 | 60 |

APの初期値は10です。

### BLD

- クトゥルフのビルドと同義
- 一般的な人はBLD 1
- 体格入力は整数

指標：

| BLD | 指標 |
|---:|---|
| -2以下 | 非常に小柄 |
| -1～0 | 小柄 |
| 1 | 標準的な人間 |
| 2 | 大柄な人間 |
| 3 | 巨体 |
| 4以上 | 人間を超える巨体 |

## 成功値と能力倍率

PCの基準成功値は50です。モンスターの基準成功値は個別入力で、初期値40です。

```text
基礎ステータス倍率 ＝ 1＋能力値÷10
最終成功値 ＝ 基準成功値×基礎ステータス倍率＋命中・技能・状況補正
```

潜在能力1以上：

```text
潜在能力倍率 ＝ 1＋潜在能力値×0.15
```

潜在能力0は使用不可です。

能力倍率、潜在能力倍率、攻撃式などの内部数値は通常画面では隠し、KPモードで表示します。最大HP、最大AP、SP、MP、FPの結果は通常画面にも表示します。

## 武器

武器は2本まで登録できます。

### 武器2

- 「武器2あり」のチェック式
- 未チェック：入力欄を非表示
- チェックを外しても入力済み内容は保持
- 通常の完全版Markdownでは、形式統一のため必ず武器2見出しを残し、未使用なら`なし`
- キャラクターシート要約では、未使用の武器2を表示しない
- KPシミュレーションでは、未使用の武器2を選択不可

### 武器入力

- 武器名
- 武器攻撃型
- 単発威力
- 命中に使う能力
- 武器説明

命中に使える能力：

- 筋力
- 技量
- 敏捷
- 感覚
- 知性
- 精神

知性・精神も攻撃成功値とKP戦闘シミュレーションへ反映します。

### 武器攻撃型

| 攻撃型 | 武器カテゴリの目安 | 基準単発威力 | レギュラー成功 |上位成功時の最大命中数 |
|---|---|---:|---:|---:|
| 一撃型 | 大剣、大斧、大槌 | 18 | 1 | 1 |
| 重撃型 | 両手剣、戦斧、長柄武器 | 10 | 1 | 2 |
| 標準型 | 片手剣、槍、棍棒 | 7 | 2 | 3 |
| 連撃型 | 軽剣、曲剣、短剣 | 6 | 3 | 4 |
| 多段型 | 双剣、格闘、爪 | 5 | 4 | 5 |

武器カテゴリは選択時の目安であり、武器名を特定の型へ固定するルールではありません。

武器攻撃型を変更すると、その型の基準単発威力を自動入力します。その後はKPから伝えられた値へ変更できます。

### 単発威力の指標

選択中の攻撃型の基準威力に対する比率で判定します。

| 条件 | 指標 |
|---|---|
| 威力0以下 | 威力なし |
| 基準の75％未満 | 簡素な武器 |
| 基準の75％以上、125％未満 | 駆け出しの武器 |
| 基準の125％以上、175％未満 | 一人前の武器 |
| 基準の175％以上、250％未満 | 熟練者の武器 |
| 基準の250％以上 | 達人級の武器 |

### 武器の出力

```text
鉄の剣【7】
```

角括弧内は**単発威力**です。武器説明などは完全版Markdownで別行に出力します。

武器が2本ある場合の要約：

```text
武器：鉄の剣【7】　短剣【4】
```

`武器1：`や`武器2：`は付けません。

- 防護点
- 防具説明
- 防護点の初期値：2
- 防御に使う装備分類欄は通常画面に置かない
- 防護点は一命中ごとに減算

防護点の指標：

| 防護点 | 指標 |
|---:|---|
| 0以下 | 無防護 |
| 1 | 簡素な防具 |
| 2 | 駆け出し防具相当 |
| 3～4 | 一人前手前の防御力 |
| 5 | 標準防御力 |
| 6～7 | 高い防御力 |
| 8以上 | 非常に高い防御力 |

防具の出力：

旅装【2】

角括弧内は**防護点**です。

## スキル

### 入力順

1. 気力・魔力・聖力のいずれかへポイントを割り振る
2. スキルを追加
3. 使用する潜在能力と消費を先に選択
4. 威力、命中、対象、範囲、追加効果を入力

消費を決めるまで、ほかの性能入力欄は使用できません。

スキル入力：

- スキル名
- 使用する潜在能力
- 威力表示
- 命中補正
- 消費
- 対象指定
- 最大対象数（複数指定だけ）
- 範囲形状と距離・角度・半径（範囲指定だけ）
- 追加効果予算
- 効果文章

スキルの出力：

```text
突進【35】
```

角括弧内はスキルの**威力表示**です。「威力」という文字は角括弧内へ追加しません。

### 性能予算

消費1につき性能予算65です。

| 消費 | 性能予算 |
|---:|---:|
| 1 | 65 |
| 2 | 130 |
| 3 | 195 |
| 4 | 260 |
| 5 | 325 |
| 10 | 650 |

消費プルダウンには`気力1（性能予算65）`のように予算を表示します。

現在採用している計算式：

```text
威力係数 ＝ 1＋威力表示÷100
命中係数 ＝ max（0.40, 1＋命中補正÷50）
性能指数 ＝ 威力係数×命中係数
基本性能差 ＝ 性能指数－1
対象予算 ＝ 複数指定のとき（最大対象数－1）×25、その他は0
追加効果負荷 ＝ 追加効果予算÷100
性能負荷合計 ＝ 基本性能差＋（対象予算＋範囲予算）÷100＋追加効果負荷
必要消費 ＝ ceil（max（0, 性能負荷合計）÷0.65）
```

画面では0.65を65に換算し、次を表示します。

- 性能予算
- マイナス補正による予算増加
- 使用中予算
- 残り予算
- 必要消費

威力または命中へマイナス補正を付け、基本性能差が負になった場合、そのぶん使用可能な予算が増えます。

- 威力表示の下限：-100
- 命中係数の下限：0.40
- 選択消費の予算を超える設定変更は受け付けない
- 必要消費より小さい消費のスキルは完成扱いにしない

強力で尖ったスキルも、計算式に収まっていれば登録できます。

### 追加効果

- 追加効果予算：0～999
- 任意入力
- 効果文章があっても予算0でよい
- 必要な場合はKPと相談して予算を決定
- 「追加効果段階」は使用しない
- 「迎撃に使用可能」の専用項目は使用しない
- 効果文章があり予算0でも、コピー時に警告しない

## 対象指定と範囲

対象指定：

- 単体
- 複数指定
- 範囲指定

### 単体

- 対象予算0

### 複数指定

```text
対象予算 ＝（最大対象数－1）×25
```

| 最大対象数 | 対象予算 |
|---:|---:|
| 2体 | 25 |
| 3体 | 50 |
| 4体 | 75 |
| 5体 | 100 |

- 最大対象数は2～99体
- プルダウンに目安を表示
- 数値欄へ自由入力も可能

### 範囲指定

範囲指定には最大対象数を入力しません。範囲内にいる対象すべてを候補として扱います。

共通下限：

- 距離と半径：1m以上
- 距離と半径：0.5m刻み
- 扇状の角度：15°以上
- 通常の範囲予算：最低5
- 地点範囲予算：最低10

#### 直線

```text
直線予算 ＝ max（5, ceil（距離÷3×15））
```

- 3m：15

#### 扇状

```text
扇状予算 ＝ max（5, ceil（20×角度÷90×（距離÷3）の2乗））
```

- 90°・3m：20
- 角度と距離の両方を設定

#### 周囲

```text
周囲予算 ＝ max（5, ceil（20×（半径÷1.5）の2乗））
```

- 360°として扱う
- 半径1.5m：20

#### 地点範囲

```text
地点範囲予算 ＝ max（10, ceil（指定可能距離÷3×10＋20×（範囲半径÷1.5）の2乗））
```

- 使用者から地点までの指定可能距離を設定
- 指定地点からの範囲半径を別に設定

距離、角度、半径はプルダウンで目安を確認しつつ、下限以上の数値を自由入力できます。

## KPモード

ページ左下の合言葉欄へ日本語で入力し、チェックマークを押すとKPモードへ入ります。

- 合言葉は伏字にしない
- 合言葉欄は左下
- 成功時：`星の裏側へ接続しました。`
- 終了時：`通常の星明かりへ戻ります。`
- 合言葉そのものは公開用ルール文書へ記載しない

KPモードで表示・使用できるもの：

- 内部数値
- 基礎ステータス倍率
- 潜在能力倍率
- 実務技能補正
- 最大HP式
- 最大AP式
- 攻撃式
- 武器受けカテゴリと倍率
- スキルの性能指数、性能負荷、性能予算、必要消費
- 自由入力した命中・判定式とダメージ式
- 自由式攻撃試験
- 対ゴブリン判定シミュレーション
- KP内部設定を含むMarkdownコピー

合言葉は通常利用者からUIを隠すための簡易ロックです。静的サイトなので、サーバー認証のような完全な秘密保持機能ではありません。

## 自由式試験

使用可能：

- 能力名
- `NdM`形式のダイス
- `+`、`-`、`*`、`/`
- 括弧
- `<`、`<=`、`>`、`>=`、`==`、`!=`

制限：

- ダイス：1～100個
- 面数：2～10000
- 0除算は不可
- 不明な能力名は不可

例：

```text
1d100 <= 技量 * 5
2d6 + 筋力
```

## 対ゴブリン判定シミュレーション

### 通常ゴブリン

- 基準成功値：40
- HP：25
- 筋力：0
- 技量：0
- 敏捷：1
- 感覚：1
- 武器：標準型、単発威力7
- 防護点初期値：1
- 武器受け倍率初期値：0.95

### PCの攻撃方法

| 攻撃方法 | 威力表示補正 | 命中補正 |
|---|---:|---:|
| 慎重な攻撃 | -10 | +5 |
| 通常攻撃 | 0 | 0 |
| 強攻撃 | +20 | -10 |
| 設定したスキル攻撃 | スキル設定値 | スキル設定値 |

### d100成功段階

| 段階 | 条件 |
|---|---|
| クリティカル | クリティカル上限以下 |
| イクストリーム成功 | 成功値の1/5以下 |
| ハード成功 | 成功値の1/2以下 |
| レギュラー成功 | 成功値以下 |
| 失敗 | 成功値を超える |

成功値100未満のクリティカル上限は1です。成功値100以上では、100を超えたぶんクリティカル上限が上昇します。成功値100以上は自動成功として扱います。

攻撃成功段階倍率：

| 段階 | 倍率 |
|---|---:|
| クリティカル | 2.50 |
| イクストリーム成功 | 1.30 |
| ハード成功 | 1.15 |
| レギュラー成功 | 1.00 |
| 失敗 | 0 |

一命中の基礎ダメージ：

```text
武器単発威力
× 攻撃方法またはスキルの威力係数
× 潜在能力倍率
× 成功段階倍率
```

最終ダメージは、防御結果倍率、武器受け倍率、防護点、命中数を反映します。防護点は一命中ごとに減算します。

### ゴブリンの防御方法

| 防御方法 | 参照能力 | 処理 |
|---|---|---|
| 防御 | 筋力 | 成功段階に応じて損害を軽減し、武器受け倍率を適用 |
| 受け流し | 技量 | 成功すれば攻撃を阻止 |
| 回避 | 敏捷 | 成功すれば攻撃を阻止 |
| 迎撃 | 技量 | d100を2回振り、悪い方を採用して対抗 |

防御成功時の損害倍率：

| 防御成功段階 | 倍率 |
|---|---:|
| クリティカル | 0.20 |
| イクストリーム成功 | 0.35 |
| ハード成功 | 0.50 |
| レギュラー成功 | 0.70 |
| 失敗 | 1.00 |

迎撃：

- ゴブリンはd100を2回振り、悪い方を採用
- 片方だけ成功：成功した側が命中
- 双方成功：小さい出目が勝利
- 同じ出目：武器衝突として双方のHP損害0

シミュレーションは1～1000回です。平均損害、一撃戦闘不能率、15％負傷級、40％負傷級以上、迎撃によるPC側損害などを表示します。

## Markdown出力

### プレイヤー用

- `■`などの記号表示を中心に出力
- `5/15`のような内部数値を表示しない
- KP内部設定を含めない

### チャッピー用

- 記号表示に加えて内部数値と上限を`（5/15）`形式で併記
- KP内部設定を含めない

### KP設定用

- プレイヤー情報
- 内部倍率
- 武器受け
- スキル計算
- 判定式
- シミュレーション設定

を含めます。

すべての完全版出力に含む内容：

- 作成対象
- 名前
- 合計、使用、残りポイント
- 体格
- 基礎ステータス
- 潜在能力
- キャラクターの場合は実務技能
- 最大HP
- 最大AP
- SP、MP、FP
- 武器
- 防具
- スキル
- 固有能力
- キャラクター設定

空の自由入力欄は`なし`と出力します。

武器2未使用時も、完全版の書式を人物ごとに変えないため次の形を残します。

```text
### 武器2

なし
```

- プレイヤー用Markdownから再読込可能
- チャッピー用Markdownから再読込可能

- KP設定マーカー：V6
- 合言葉は出力しない

## 自動保存

- 入力変更ごとにブラウザの`localStorage`へ保存
- ページ再読込後も復元
- PCでも再読込可能
- 保存キー：`hoshimichi-character-sheet-builder`
- 現在の保存データバージョン：7
- バージョン1～6をバージョン7へ移行

旧データ移行：

- 武器2に実入力があれば「武器2あり」
- 武器2が初期値のままなら「なし」
- 旧自由入力スキル・武器・装備は移行用メモとして保持
- 旧構造化武器は武器1へ移行
- 旧スキルの追加効果予算は0として移行
- 旧対象形状を単体・複数指定・範囲指定へ移行

壊れた保存データや未対応バージョンを検出した場合は、画面を停止せず初期値へ戻します。
