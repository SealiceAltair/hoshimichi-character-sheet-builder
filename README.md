# 星みちTRPG キャラクタービルダー

星みちTRPGのキャラクター作成時に、共通ポイントを基礎ステータス・潜在能力・実務技能へ配分し、装備・スキル・キャラクター設定をまとめたMarkdownを作るための試作キャラクタービルダーです。

このフォルダだけで動作します。GitHub、サーバー、ビルド、外部ライブラリ、外部フォントは不要です。入力内容を外部へ送信する処理もありません。

会話内で決定し、現在のサイトへ実装したルールと表示記号の読み方は、[RULES.md](RULES.md)へまとめています。特に`★・☆・■・△`の換算で迷った場合は、RULES.mdの冒頭を確認してください。

武器攻撃型・強化・防護の数値は、非公開正本`355_星みちTRPG_武器攻撃型・強化・防護.md`に同期しています。

## 起動方法

1. \`index.html\` をダブルクリックします。
2. Webブラウザで画面が開いたら、そのまま入力します。

ローカルファイルとして直接開けるほか、同じファイル構成を任意の静的ホスティングへ配置しても動作します。

## 入力項目

- 基本情報
  - 名前：コピー時の必須項目
  - 作成対象：キャラクターまたはモンスター
  - 合計ポイント：初期値4、0～999。40は「銀級上位」の指標
  - 体格：クトゥルフの「ビルド」と同義。一般的な人の初期値1を「標準的な人間」と表示
- 基礎ステータス
  - 頑健、持久、筋力、技量、敏捷、感覚、知性、精神
- 潜在能力
  - 気力：肉体強化、武技。みんな持ってる。正しく使えるのは一握り
  - 魔力：魔術、錬金術。10人にひとりの才能
  - 聖力：治癒、浄化、加護。100人にひとりの才能
- 実務技能（キャラクターのみ）
  - 料理、整備、手当、植物、鉱物、解体、冒険
- HP・武器・防具
  - PC最大HPは \`50×（1＋頑健÷3）\` を四捨五入
  - 最大APは \`10×（1＋持久÷3）\` を四捨五入。持久0で10、持久15で60
  - モンスターは基準成功値と個別HPを入力
  - 武器1と、チェック式で使用する武器2に、武器名、攻撃型、基礎単発威力、強化段階、命中に使う能力、武器説明
  - 強化後単発威力を自動表示
  - 武器の命中には筋力、技量、敏捷、感覚、知性、精神を指定可能
  - 防具名、防護点、防具説明。防護点の初期値は2
- 構造化スキル
  - スキル名、潜在能力種別、威力、命中補正、消費、対象指定、最大対象数、範囲形状、距離・角度・半径、追加効果予算、効果
- ポイントを消費しない別枠（基本は入力不要。必要な場合のみ入力）
  - 固有能力
- キャラクター設定
  - 性格、外見、経歴、話し方などを自由入力
- キャラクターシート要約
  - 名前、武器、防具、スキル、CP、HP、AP、BLD、SP・MP・FP、割り振り能力を自動作成してコピー

ポイント配分の数値項目は \`－\` ボタン、数値入力欄、\`＋\` ボタンのいずれでも変更できます。小数、負数、上限を超えた値は許可範囲内の整数へ補正されます。体格も整数で入力し、小数は整数へ補正されます。

## ポイント計算

使用ポイントは、次の合計です。

\`\`\`text
基礎ステータス合計
＋ 潜在能力合計
＋ 実務技能合計
\`\`\`

上記は作成対象がキャラクターの場合です。モンスターの場合、実務技能は画面に表示せず、使用ポイントの計算からも除外します。切替前に入力した実務技能値は削除せず保持するため、キャラクターへ戻すと再び表示・計算されます。

残りポイントは「合計ポイント－使用ポイント」です。1上昇につき1ポイントを消費します。

- 残りポイントが0になると、それ以上の加算を拒否します。
- 合計ポイントを使用ポイント未満へ変更しても、配分済みの値は自動削除しません。
- 使用ポイントが合計を超えた場合は、超過数と警告を表示します。
- ポイント超過中はMarkdownのコピーボタンを無効化します。

## 各項目の上限

| 区分 | 項目 | 上限 |
|---|---|---:|
| 基礎ステータス | 頑健、持久、筋力、技量、敏捷、感覚、知性、精神 | 各15 |
| 潜在能力 | 気力、魔力、聖力 | 各10 |
| 通常の実務技能 | 料理、整備、手当、植物、鉱物、解体 | 各3 |
| 特別な実務技能 | 冒険 | 5 |

実務技能の段階名は、0が「未習得」、1が「心得あり」、2が「得意」、3が「熟練」です。冒険の4と5には固定の段階名を付けていません。

実務技能はキャラクター用です。特に冒険は、プレイヤーとチャッピーの共同表現へ作用するため、NPCやモンスターには使用しません。薬草を扱うゴブリンシャーマンや鍛冶を行うモンスターなど、例外的な能力は別枠の「スキル」へ個別登録します。

\`\`\`text
粗雑な薬草調合
武器修繕
\`\`\`

## 表示

- 基礎ステータスの0は「一般人」と表示します。
- 潜在能力と実務技能の0は「―」と表示します。
- 1ポイントにつき「■」を1個表示します。
- 5個ごとに半角スペースで区切ります。

内部の計算と保存には整数値を使用します。

基礎ステータス欄には、その能力が主に関わる判定と段階指標を表示します。潜在能力には、正本に基づく説明と保有の目安、および「使用不可」または「習得数1・1日1回」の形式で習得数と一日の使用可能回数を表示します。気力0は一般基準を超えるポイントを配分していない状態、魔力0・聖力0は人物正本に例外がない限り現在使用可能な潜在能力が確認されていない状態です。合計ポイントと体格にも指標を表示し、合計40は固有名を出さず「銀級上位」と表示します。通常画面では能力倍率・潜在能力倍率・攻撃式を表示しません。最大HPは通常画面でも表示します。能力欄の上限を示す分数表示は行いません。

## 武器・防具

武器は2本まで入力できます。武器2は「武器2あり」のチェックを入れたときだけ入力欄を表示します。チェックを外しても入力値は保持しますが、通常出力の武器2欄は形式統一のため「なし」、要約では武器2自体を省略します。武器攻撃型を選ぶと、現在採用している単発威力の基準値が入ります。その後、KPから伝えられた値へ変更できます。単発威力は選んだ攻撃型の基準値と比較し、「簡素な武器」「駆け出しの武器」「一人前の武器」「熟練者の武器」「達人級の武器」の指標を表示します。同じ単発威力でも、攻撃型によって指標が変わります。

| 攻撃型 | 武器カテゴリの目安 | 基礎単発威力 |
|---|---|---:|
| 重撃型 | 両手剣、戦斧、長柄武器 | 18 |
| 一撃型 | 大剣、大斧、大槌 | 13 |
| 標準型 | 片手剣、槍、棍棒 | 7 |
| 連撃型 | 軽剣、曲剣、短剣 | 6 |
| 多段型 | 双剣、格闘、爪 | 5 |

武器カテゴリは選択しやすくするための代表例であり、武器名をその型へ固定するルールではありません。武器の戦い方や想定する命中回数に合う型を選びます。

武器強化段階は0～3です。`強化後単発威力＝基礎単発威力×強化倍率＋固定値`で計算し、段階0は`×1＋0`、段階1は`×4/3＋0.50`、段階2は`×5/3＋1.50`、段階3は`×2＋2.00`です。

Markdown要約では強化0の武器名はそのまま、強化1～3は`武器名＋N`と表示します。防具名へ防護点を角括弧で付けません。スキルの`スキル名【威力表示】`形式は変更しません。

武器受けカテゴリと倍率は武器ごとに持ち、KPモードだけで設定・表示します。武器2へ盾を登録することもできます。防具の防護点は一命中ごとに減算します。防護点2は「駆け出し防具相当」、防護点5は「標準防御力」の指標です。PC防具は防護点を直接入力し、敵用の防護強化式を自動適用しません。

## スキル設定

潜在能力値は、その潜在能力の習得可能数と一日の使用可能量を兼ねます。潜在能力0ではスキルを追加できません。

1. 気力・魔力・聖力のいずれかに習得枠を用意します。
2. スキルを追加します。
3. 潜在能力種別と消費を先に選択します。
4. 威力、命中補正、対象、範囲、効果を入力します。

消費プルダウンには「気力1（性能予算65）」のように、消費ごとの性能予算を表示します。

必要消費は次の現在採用式で自動計算します。

\`\`\`text
威力係数 ＝ 1＋威力表示÷100
命中係数 ＝ max（0.40, 1＋命中補正÷50）
性能指数 ＝ 威力係数×命中係数
基本性能差 ＝ 性能指数－1
対象予算 ＝ 複数指定のとき（最大対象数－1）×25、その他は0
直線予算 ＝ max（5, ceil（距離÷3×15））
扇状予算 ＝ max（5, ceil（20×角度÷90×（距離÷3）の2乗））
周囲予算 ＝ max（5, ceil（20×（半径÷1.5）の2乗））
地点範囲予算 ＝ max（10, ceil（指定可能距離÷3×10＋20×（範囲半径÷1.5）の2乗））
追加効果負荷 ＝ 追加効果予算÷100
性能負荷合計 ＝ 基本性能差＋（対象予算＋範囲予算）÷100＋追加効果負荷
必要消費 ＝ ceil（max（0, 性能負荷合計）÷0.65）
\`\`\`

画面では0.65を65へ換算し、「性能予算」「マイナス補正による予算増加」「使用中」「残り」として表示します。消費1の性能予算は65、消費2は130、消費3は195です。威力または命中のマイナス補正で基本性能差が負になった場合、その分だけ残り予算が増えます。追加効果予算は段階表から選ばず、必要な場合にKPと相談して決めた0～999の整数を直接入力します。追加効果予算は任意で、効果欄に文章があっても0のままコピーできます。選択した消費の性能予算を超える変更は受け付けません。潜在能力値を、習得済みスキル数または設定済み消費より低くすることもできません。

対象指定は「単体」「複数指定」「範囲指定」から選びます。複数指定だけが最大対象数を使い、2体は対象予算25、3体は50、4体は75、5体は100です。最大対象数は目安プルダウンだけでなく、2～99体の自由入力もできます。

範囲指定は「直線」「扇状」「周囲」「地点範囲」から選びます。直線3mは範囲予算15、扇状90°・3mは20、周囲半径1.5mは20です。地点範囲は、指定可能距離と着弾地点からの範囲半径を別々に設定します。範囲指定には最大対象数を入力せず、その形状内にいる対象すべてを候補として扱います。距離と半径は1m以上を0.5m刻み、扇状の角度は15°以上で入力します。通常の範囲予算は最低5、地点範囲は最低10です。プルダウンで予算目安を確認でき、数値欄では下限以上の任意値を自由入力できます。この安全下限を満たす範囲では、強力で尖ったスキルも登録できます。

## 自動保存

入力変更ごとにブラウザの \`localStorage\` へ自動保存します。ページを閉じたり再読込したりしても、同じブラウザであれば入力内容を復元します。

保存キーは \`hoshimichi-character-sheet-builder\`、保存データのバージョンは \`8\` です。バージョン1～7の保存データは、PC・モンスターの入力内容を維持したままバージョン8へ読み替えます。V7以前の武器には強化段階0、KP設定には敵防護段階0を補います。旧データの武器2に実入力があれば「武器2あり」として移行し、初期値のままなら「なし」として移行します。バージョン4以前のスキルは追加効果予算0として引き継ぎ、バージョン5以前の対象形状は「単体」「複数指定」「範囲指定」と新しい範囲項目へ移行します。旧版の自由入力スキル・武器・装備は移行用メモとして保持し、旧版の構造化武器は武器1へ移行します。壊れたデータや未対応バージョンを検出した場合は、画面を停止せず初期値へ戻します。

ブラウザのプライベートモードや保存禁止設定では、自動保存できない場合があります。その場合も画面上の入力と計算は利用できます。

## コピー機能

### プレイヤー用Markdownをコピー

「■」を中心とした表示をコピーします。各能力値の \`7/15\` などの内部数値は含めません。

### チャッピー用データをコピー

「■」表示に加え、各項目の内部数値と上限を \`（7/15）\` の形式で併記します。

両方のコピーに、作成対象、名前、ポイント状況、体格、最大HP、最大AP、対象に応じた能力値、固有能力、構造化スキル、武器、防具、キャラクター設定を含めます。モンスターでは実務技能を出力しません。空の自由入力欄は「なし」と出力します。

### キャラクターシート要約をコピー

画面最下部の要約は入力内容から自動作成します。CP・HP・AP・正のBLDは \`★＝5\`、\`☆＝1\` とし、5記号ごとに空白で区切ります。BLD 0は \`△\`、BLD－1は \`△△\`、BLD－2は \`△△△\` のように、負側は短縮せず表示します。SP・MP・FPと割り振り能力は \`■＝1\` です。BLDはAPの直下に置き、BLD以外の0の資源・能力、名前のない武器・防具・スキルは表示しません。武器が2本ある場合も番号を付けず、同じ行へ横並びで出力します。

コピー内容の末尾には、設定の再読込に使用するHTMLコメントを付けます。通常のMarkdown表示では見えません。PC・モンスターの設定だけを含み、KPモードの判定式・倍率・合言葉は含みません。

名前が未入力の場合はコピーせず、名前欄へ移動して警告します。ブラウザのクリップボード機能を使用できない場合は、旧方式のコピーまたは手動コピー画面へ切り替わります。

## KPモード

ページ最下部の左側で合言葉を入力し、チェックマークを押すとKPモードを開きます。合言葉は日本語の通常文字として表示されます。接続成功時は「星の裏側へ接続しました。」、終了時は「通常の星明かりへ戻ります。」と表示します。

KPモードでは次を使用できます。

- 基礎ステータス倍率、潜在能力倍率、実務技能補正
- 基準成功値、最大HP式、最大AP式、武器受け倍率、防護点
- 構造化スキルの性能指数・性能負荷・性能予算・必要消費
- 攻撃式と防御式
- 命中・判定式とダメージ式
- 1～100回の攻撃試験
- 通常ゴブリンを相手にした1～1000回の判定シミュレーション
- KP内部設定を含むKP設定Markdownのコピー

自由式判定では能力名、\`NdM\`形式のダイス、四則演算、括弧、\`<\`、\`<=\`、\`>\`、\`>=\`、\`==\`、\`!=\`を使用できます。能力名は入力済みの生値として参照します。ダイスは1～100個、2～10000面までです。

例：

\`\`\`text
1d100 <= 技量 * 5
2d6 + 筋力
\`\`\`

KPモードの合言葉はUIを通常利用から隠すための簡易ロックです。このサイトは静的なGitHub Pagesで動作するため、サーバー認証のような完全な秘密保持機能ではありません。

### 対ゴブリン判定シミュレーション

攻撃方法は「慎重な攻撃」「通常攻撃」「強攻撃」「設定したスキル攻撃」から選びます。ゴブリン側は「防御」「受け流し」「回避」「迎撃」から選びます。

通常ゴブリンは現在採用値の基準成功値40・HP25です。基礎防護点、防護段階0～3、武器受け倍率をKPモード内で調整できます。強化後防護点は`基礎防護点×防護倍率＋固定値`で計算します。迎撃はd100を2回振って悪い方を採用し、双方成功時は小さい出目が勝ち、同値では双方のHP損害を0にします。攻撃は355の型別ヒット数・型別威力倍率・武器強化を使い、全ヒット合計後だけ四捨五入します。

## Markdownから再読み込み

通常画面の「Markdownから設定を再読み込み」へ、このサイトでコピーしたMarkdownを貼り付けます。PC・モンスターのどちらでも使用できます。

1. Markdownを貼り付けます。
2. 「読込内容を確認」を押します。
3. 名前、作成対象、ポイント、体格、KP設定の扱いを確認します。
4. 「上書きする」を押します。

プレイヤー用またはチャッピー用Markdownでは、PC・モンスター、HP、武器、防具、構造化スキルの設定を上書きし、現在のKP内部設定は維持します。KP設定Markdownでは、これらに加えて判定式・ゴブリン試験設定・試験回数も上書きします。旧形式のMarkdownは、見出しと「■」または内部数値を読み取れる場合に限り復元します。

## リセット方法

「すべてリセット」を押すと確認ダイアログが表示されます。確認後、次を初期化します。

- 名前、すべての数値（体格は1へ戻る）
- 固有能力、構造化スキル、武器、防具、キャラクター設定、旧版移行メモ
- 合計ポイント（4へ戻る）
- ブラウザ内の保存データ

確認を取り消した場合は何も削除しません。

## ルール検証

Node.jsを利用できる環境では、依存関係を追加せずに355数値、武器・防護強化、V8移行条件、要約表記、最終丸めを検査できます。

\`\`\`powershell
node test-rules.js
\`\`\`

## 注意

このツールは試作であり、星みちTRPGのルール正本ではありません。ここで使用するステータスルールや入力内容は、正本へ自動反映されません。

正本の更新は、このツールとは別の確認・更新工程で行ってください。

モンスターの実務技能を非表示・計算対象外とするルールの確定度は「決定（会話内／GitHub未反映）」です。ここでいう「GitHub未反映」は、星みちTRPGの正本リポジトリへ未反映であることを指します。
